# TG Parser Bot 🤖

**Система автоматического мониторинга Telegram-чатов для генерации лидов**  

Этот бот автоматически отслеживает сообщения в групповых чатах (например, чаты ЖК), ищет потенциальных клиентов по ключевым словам и маркерам намерения, и отправляет им персонализированные сообщения с предложением услуг.

## 📋 Особенности

- ✅ **Автоматический поиск лидов** - Мониторинг чатов по триггерам и маркерам
- ✅ **Интеллектуальный анализ** - Комбинирует триггеры (что ищут) с маркерами контекста (как ищут)
- ✅ **Задержка отправки** - Рандомная задержка 5-12 минут для естественности
- ✅ **Spintax поддержка** - Вариативность текстов через `{Привет|Здравствуйте}`
- ✅ **Уведомления о реакциях** - Отправка первых ответов клиентов в рабочий чат
- ✅ **Защита от спама** - Запись обработанных пользователей
- ✅ **Простое управление** - Админ-бот через Telegram-команды

## 🏗 Архитектура

Система состоит из трёх компонентов:

1. **admin_bot.py** - Админ-панель для управления настройками
2. **userbot.py** - Парсер и автоотправка сообщений (Pyrogram)
3. **database.py** - SQLite база данных для хранения настроек и истории

## 🚀 Быстрый старт

### 1. Требования

- Python 3.9+
- Telegram API ID и API Hash (получите на https://my.telegram.org)
- Telegram Bot Token (получите у @BotFather)
- Два аккаунта Telegram:
  - **Основной аккаунт** - для запуска юзербота (парсер)
  - **Админ-аккаунт** - для управления через админ-бота (может быть тем же)

### 2. Установка

```bash
# Клонируйте репозиторий
git clone https://github.com/GeorgeFromGeorgea/tg_parser.git
cd tg_parser

# Создайте виртуальное окружение
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# или venv\Scripts\activate  # Windows

# Установите зависимости
pip install -r requirements.txt
```

### 3. Настройка

#### 3.1 Получение API данных

1. Перейдите на https://my.telegram.org/apps
2. Войдите под основным аккаунтом (для юзербота)
3. Создайте приложение и запишите:
   - `api_id` (число)
   - `api_hash` (строка)

#### 3.2 Создание бота

1. В Telegram найдите @BotFather
2. Отправьте `/newbot` и следуйте инструкциям
3. Запишите полученный `BOT_TOKEN`

#### 3.3 Узнайте ваш Telegram ID

1. В Telegram найдите @userinfobot
2. Отправьте любое сообщение
3. Запишите полученный `USER_ID`

#### 3.4 Настройка файлов

**Отредактируйте `userbot.py`:**

```python
API_ID = 1234567          # Замените на ваш api_id
API_HASH = "your_api_hash"  # Замените на ваш api_hash
```

**Отредактируйте `admin_bot.py`:**

```python
BOT_TOKEN = "your_bot_token_here"  # Замените на ваш токен от BotFather
ADMIN_ID = 111111111  # Замените на ваш USER_ID от @userinfobot
```

### 4. Запуск

#### Первый запуск (авторизация юзербота)

```bash
python userbot.py
```

При первом запуске вас попросят авторизоваться:

```
phone: +79991234567  # Введите номер телефона (с плюсом)
password: ********  # Если включена 2FA, введите код
```

После успешной авторизации создастся файл `manager_session.session` - **не удаляйте его!**

#### Запуск в продакшене

Для непрерывной работы запустите оба процесса в фоновом режиме:

```bash
# Запуск юзербота (парсер)
python userbot.py &

# Запуск админ-бота (панель управления)
python admin_bot.py &
```

**Рекомендация:** Используйте `systemd`, `screen` или `tmux` для стабильной работы.

## ⚙️ Управление через Админ-бот

После запуска админ-бота отправьте команду `/start` в Telegram, чтобы увидеть полный список команд:

### Основные команды

| Команда | Описание | Пример |
|---------|----------|--------|
| `/set_work_chat ID` | Установить рабочий чат для уведомлений | `/set_work_chat -100123456789` |
| `/add_chat ID` | Добавить чат для мониторинга | `/add_chat -100123456789` |
| `/del_chat ID` | Удалить чат из мониторинга | `/del_chat -100123456789` |
| `/chats` | Список всех отслеживаемых чатов | `/chats` |
| `/add_trigger слово` | Добавить триггер поиска | `/add_trigger плитка` |
| `/del_trigger слово` | Удалить триггер | `/del_trigger плитка` |
| `/triggers` | Список всех триггеров | `/triggers` |
| `/add_marker слово` | Добавить маркер контекста | `/add_marker посоветуйте` |
| `/del_marker слово` | Удалить маркер | `/del_marker посоветуйте` |
| `/markers` | Список всех маркеров | `/markers` |
| `/set_template текст` | Установить шаблон ответа | `/set_template {Привет|Здравствуйте}!` |

### Пример настройки

```bash
# 1. Установите рабочий чат (где будут приходить уведомления)
/set_work_chat -1001234567890

# 2. Добавьте чаты ЖК для мониторинга
/add_chat -1001111111111  # ЖК "Мир"
/add_chat -1002222222222  # ЖК "Парус"

# 3. Настройте триггеры (что ищут люди)
/add_trigger ремонт
/add_trigger плитка
/add_trigger сантехника

# 4. Настройте маркеры (как ищут)
/add_marker подскажите
/add_marker посоветуйте
/add_marker ищу мастера

# 5. Установите шаблон ответа
/set_template {Привет|Здравствуйте}! Видел ваш вопрос в чате ЖК. Мы как раз заканчиваем {объект|ремонт} в соседнем корпусе, могу подсказать по стоимости или прислать смету для примера.
```

## 🎯 Как это работает

### Алгоритм поиска лидов

1. **Мониторинг чатов** - Юзербот читает все сообщения в добавленных чатах
2. **Фильтрация** - Пропускает сообщения от ботов и без текста
3. **Проверка дублей** - Проверяет, не обрабатывали ли этого пользователя ранее
4. **Анализ текста**:
   - Ищет **триггеры** (например, "плитка", "ремонт")
   - Ищет **маркеры** (например, "подскажите", "посоветуйте")
5. **Валидация лида** - Если найдены ОБА типа слов → считаем лидом
6. **Задержка** - Ждёт 5-12 минут для естественности
7. **Отправка** - Пишет в ЛС с использованием spintax
8. **Запись** - Сохраняет пользователя в БД

### Обработка ответов клиентов

Когда клиент отвечает на сообщение юзербота:

1. Юзербот получает ответ в личные сообщения
2. Проверяет, первый ли это ответ
3. Формирует красивое уведомление с данными клиента
4. Отправляет в рабочий чат
5. Меняет статус клиента (больше не уведомляет)

### Spintax синтаксис

Для вариативности текстов используйте синтаксис:

```
{Привет|Здравствуйте|Добрый день}
{ремонт|отделку|ремонтные работы}
{могу подсказать|помогу определиться|расскажу подробно}
```

При каждом отправке будет выбираться случайный вариант.

## 🛠 Troubleshooting

### Ошибка "Invalid API ID"
- Проверьте, что используете данные **с того же аккаунта**, что будет запускать юзербот
- Получите новые данные на https://my.telegram.org/apps

### Бот не отправляет сообщения
- Проверьте, что юзербот авторизован (`manager_session.session` существует)
- Убедитесь, что API_ID и API_HASH верны в `userbot.py`
- Проверьте логи на наличие ошибок

### Нет уведомлений в рабочем чате
- Убедитесь, что юзербот **добавлен в рабочий чат** как участник
- Проверьте, что рабочий чат установлен через `/set_work_chat`
- Проверьте, что ID чата правильный (включая `-100` для супергрупп)

### Сообщения не отправляются клиентам
- Проверьте, что клиент **не заблокировал** ваш аккаунт
- Убедитесь, что настройки приватности Telegram разрешают ЛС
- Проверьте логи на ошибки отправки

## 🔄 Автоматический запуск (systemd)

Создайте файл `/etc/systemd/system/tg-parser.service`:

```ini
[Unit]
Description=TG Parser Bot
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/tg_parser
Environment="PATH=/root/tg_parser/venv/bin"
ExecStart=/root/tg_parser/venv/bin/python userbot.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Аналогично для админ-бота: `/etc/systemd/system/tg-admin.service`

```bash
# Включите автозапуск
sudo systemctl enable tg-parser.service
sudo systemctl enable tg-admin.service

# Запустите сервисы
sudo systemctl start tg-parser.service
sudo systemctl start tg-admin.service

# Проверьте статус
sudo systemctl status tg-parser.service
sudo systemctl status tg-admin.service
```
